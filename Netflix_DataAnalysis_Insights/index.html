üìë Getting The Raw Data

Netflix lets you request and download your personal viewing data. Here‚Äôs how you can do it:

  1. Go to Netflix‚Äôs "Get My Info" page and submit a data request.
  2. Confirm your request via the email Netflix sends you.
  3. Wait for the data to be processed‚Äîthis usually takes less than a day.
  4. Download the .zip file from the second email you receive, which contains your data in a structured format.

This file serves as the foundation for analyzing your viewing habits and building data-driven insights.

üïêüìÖ Both the ‚ÄòStart Time‚Äô and ‚ÄòDuration‚Äô columns are now in a proper format for analysis. Heres the comparision Before and After.

The 'Start Time' column is now properly formatted as a datetime object in UTC format.

  • Each value follows this structure:
    • YYYY-MM-DD HH:MM:SS+00:00, where:
      • YYYY-MM-DD ‚Üí The year, month, and day of the activity.
      • HH:MM:SS ‚Üí The exact time in 24-hour format.
      • +00:00 ‚Üí Indicates that the timestamp is in UTC time zone.
  • Examples:
    • 2024-06-02 03:56:00+00:00 ‚Üí Means the activity started on June 2, 2024, at 03:56 AM UTC.
    • 2025-01-09 18:10:00+00:00 ‚Üí Means the activity started on January 9, 2025, at 06:10 PM UTC.

Explanation of the 'Duration' Column

  • The 'Duration' column is correctly converted to timedelta format, which represents the duration of the activity.
  • Each value follows this structure:
    • X days HH:MM:SS, where:
      • X days ‚Üí Represents the number of full days in the duration.
      • HH:MM:SS ‚Üí Represents the duration in hours, minutes, and seconds.
  • Examples:
    • 0 days 00:00:04 ‚Üí The activity lasted for 4 seconds.
    • 0 days 00:42:15 ‚Üí The activity lasted for 42 minutes and 15 seconds.

Step 1: Understanding the Calculation

Each row in the dataset represents a viewing interaction, meaning every time a user watched something (a movie, an episode, or even a trailer), it's logged.

To find the average viewing time per interaction, the calculation used is:

Average Watch Time = Total Watch Duration for Profile / Total Interactions for Profile

Example:

netflix.loc[netflix['Profile Name'] == 'Jobit', 'Duration'].sum() / 1835

Explanation:

  • netflix.loc[netflix['Profile Name'] == 'Jobit', 'Duration'] ‚Üí This gets the total watch time for the "Jobit" profile.
  • 1835 ‚Üí This is the total number of interactions (views) for "Jobit".
  • Division gives the average watch time per interaction.

Step 2: Why Are the Average Watch Times So Low?

Initially, the calculated average viewing times are less than 30 minutes per interaction. This seems strange because most people watch entire episodes (usually 20-40 mins) or movies (90+ mins).

Possible Reasons:

  • Netflix logs each episode separately: If a person is watching a series marathon, each episode is counted as a separate interaction. This means the total viewing duration is split across multiple interactions.
  • Short interactions are included in the data: Netflix logs teasers, trailers, hooks, and promotional clips. These are usually a few seconds long, which reduces the average watch time.

Step 3: Filtering Out Short Interactions

To get a more accurate estimate of the actual viewing time, we need to remove short interactions like teasers, trailers, and hooks.

Key Takeaways

  • Before Filtering: The average watch time per interaction was misleadingly low.
  • After Filtering: The numbers increased, giving a more realistic estimate of actual viewing behavior.
  • Profiles that browsed a lot (e.g., "Mom & Dad") showed the biggest increase in average watch time.
  • This confirms that short interactions (trailers, hooks) had skewed the original data.

Summary

  • Filtering out short interactions was necessary to get an accurate representation of viewing habits.
  • Profiles with more trailers/hooks logged had artificially lower watch times.
  • After cleaning the data, average watch times became more reflective of real viewing behavior.
  • This method improves the accuracy of Netflix interaction analysis for real user behavior.
